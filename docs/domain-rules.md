# 도메인 규칙

## 문제

문제는 다음 정보를 가진다.

- 플랫폼
- 외부 문제 ID
- 제목
- URL
- 난이도
- 태그

### 문제 상태

문제 상태는 다음 값을 가진다.

- `ACTIVE`: 복습 대상이 될 수 있는 기본 상태
- `MASTERED`: 마스터 처리되어 복습 큐에서 제외되는 상태

`MASTERED`는 수동으로 표시하지 않고 연속 성공 횟수를 기준으로 자동 처리한다.
성공으로 보는 풀이 결과가 2번 연속 기록되면 문제는 `MASTERED`가 된다.

### 중복 등록 기준

중복 문제 등록 기준은 `platform`과 `externalProblemId`의 조합이다.
같은 문제를 여러 플랫폼에 중복 등록하지 않는다.

### 플랫폼

허용되는 플랫폼 초안은 다음과 같다.

- `BAEKJOON`
- `PROGRAMMERS`
- `LEETCODE`
- `ETC`

### 난이도

난이도는 MVP에서 자유 텍스트로 저장한다.

## 풀이 시도 결과

허용되는 풀이 시도 결과는 다음과 같다.

- `SOLVED_WITHOUT_HINT`
- `SOLVED_WITH_HINT`
- `FAILED`
- `RETRY_NEEDED`

성공으로 보는 결과는 다음과 같다.

- `SOLVED_WITHOUT_HINT`
- `SOLVED_WITH_HINT`

실패 또는 재시도로 보는 결과는 다음과 같다.

- `FAILED`: 실패 종료
- `RETRY_NEEDED`: 시간이 지나서 풀었거나 다시 풀 필요가 있는 상태

`SOLVED_WITH_HINT`의 힌트 범위는 공식 풀이, 블로그, AI, 친구 도움 등 외부 힌트를 본 경우로 넓게 정의한다.

`mistakeReason`과 `memo`는 MVP에서 자유 텍스트로 저장한다.

## 복습 일정

초기 단순 규칙은 다음과 같다.

- `FAILED`: 1일 뒤 복습
- `SOLVED_WITH_HINT`: 2일 뒤 복습
- `RETRY_NEEDED`: 3일 뒤 복습
- `SOLVED_WITHOUT_HINT`: 7일 뒤 복습

`nextReviewAt`은 사용자 로컬 날짜 기준으로 계산한다.

풀이 시도가 없는 문제는 `nextReviewAt`을 `null`로 둔다.
이 문제는 오늘의 복습 큐에 노출하지 않는다.

성공한 경우 점진적으로 복습 간격을 늘린다.
현재 구현 기준은 성공 시 이전 복습 간격의 2배와 해당 결과의 기본 간격 중 더 큰 값을 사용한다.

예:

- 첫 `SOLVED_WITHOUT_HINT`: 7일 뒤
- 두 번째 연속 `SOLVED_WITHOUT_HINT`: 14일 간격으로 증가하고 `MASTERED` 처리

실패 또는 재시도 결과가 기록되면 연속 성공 횟수는 0으로 초기화한다.

## 오늘의 복습 큐

다음 조건을 모두 만족하면 문제는 오늘의 복습 큐에 나타나야 한다.

- `nextReviewAt`이 오늘이거나 오늘보다 이전이다.
- 문제가 `MASTERED` 상태가 아니다.

## 태그

태그는 사용자가 직접 입력한다.
MVP에서는 태그 별칭을 지원하지 않는다.

태그 저장 규칙은 다음과 같다.

- 입력값 앞뒤 공백은 제거한다.
- 대소문자는 표시값으로 유지한다.
- 같은 문제에는 같은 태그 이름을 중복 저장하지 않는다.
- 별도 태그 마스터 테이블 없이 문제별 태그 이름을 저장한다.

## 삭제 정책

문제를 삭제하면 연결된 풀이 시도와 태그도 함께 삭제한다.

## 사용자 모델

MVP에서는 단일 사용자 기준으로 모델링한다.
인증과 권한은 비범위다.

## 취약 태그 통계

취약 태그 통계는 현재 복습 대상 분석으로 본다.
`weaknessScore`는 최근 시도에 가중치를 주고 풀이 시간까지 반영하는 방향으로 계산한다.
